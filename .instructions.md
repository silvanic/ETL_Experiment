# Skill: Mettre à jour la documentation

## Analyse et mise à jour automatique de la documentation

Synchronise automatiquement la documentation technique avec le code source pour détecter et corriger les divergences.

## Utilisation rapide

```
/update-etl-docs
```

## Qu'est-ce que ça fait?

1. 🔍 **Analyse le code** — Types de nœuds, composants, services, exécuteurs
2. 📊 **Compare avec la doc** — Détecte divergences avec `docs/technique/`
3. 📝 **Propose changements** — Liste les mises à jour recommandées
4. ✏️ **Applique automatiquement** — Si vous approuvez

## Quand l'utiliser?

✅ Après ajout d'un nouveau type de nœud  
✅ Après création d'un service ou composant  
✅ Après renommage de fichiers  
✅ Maintenance régulière (1x/semaine en dev actif)

## Options

```
/update-etl-docs --no-apply     # Rapport seulement
/update-etl-docs --verbose      # Logs détaillés
/update-etl-docs --focus=types  # Analyser seulement types
```
